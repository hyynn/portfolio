Design Decisions

주요 디자인 결정과 그 이유

섹션 흐름 재설계

기존 기능 나열 구조를 공감 → 솔루션 → 신뢰 → 행동 순으로 바꿨습니다. 방문자가 제품에 공감하기 전에 기능 설명이 먼저 등장하는 문제를 해결했습니다.

탭 UI + Bento Grid

기능 8개를 3섹션에 반복 나열하던 구조를 탭 UI로 통합하고, Bento Grid로 카드 크기에 정보의 중요도를 담았습니다.

배경색 3단계 전략

흰 배경을 기본으로 bgC와 mainC를 각 2~3곳에만 배치했습니다. 배경 변화로 섹션을 구분하면서도 시각적 피로를 줄였습니다.

수치 중심 설득

텍스트 설명보다 수치가 공감을 빠르게 유발한다고 판단해 Stats Banner와 Cases 섹션에 수치를 전면 배치했습니다. 카운트업 애니메이션으로 시선을 집중시켰습니다.

Hero

CSS Grid로 이미지와 텍스트를 같은 셀에 겹쳐 position: absolute 없이 풀스크린 Hero를 구현했습니다. 페이지 진입 시 자연스러운 페이드인 애니메이션이 실행됩니다.

Stats Banner

Intersection Observer로 섹션 진입 시 카운트업 애니메이션이 1회 실행됩니다. ch 단위로 숫자 너비를 고정해 뷰포트 크기 변경 시 단위 정렬이 틀어지는 문제를 해결했습니다.

Solutions

카드 hover 시 그라디언트 오버레이가 페이드인됩니다. 터치 디바이스에서는 클릭으로 오버레이를 토글하도록 분기 처리했습니다.

Features - Tab UI + Bento Grid

WAI-ARIA 속성을 적용한 탭 UI와 크기가 다른 카드로 구성한 Bento Grid를 결합했습니다. 탭 전환 시 카드가 100ms 간격으로 순차 페이드인됩니다.

Cases

업종별 도입 효과를 수치 중심으로 구성했습니다. 텍스트 설명보다 수치 하나가 더 빠르게 신뢰를 전달한다고 판단해 case-stat을 카드의 핵심 요소로 배치했습니다.

Process + Trust Numbers

3단계 프로세스 하단에 신뢰 수치 배너를 배치해 "이렇게 쉽게 시작할 수 있고, 이미 이만큼 많은 매장이 선택했다"는 흐름이 자연스럽게 이어지도록 구성했습니다.

Header - GNB 활성 표시

Intersection Observer로 현재 섹션을 감지해 해당 GNB 링크에 active 클래스를 토글합니다. 스크롤 이벤트 대신 Observer를 사용해 성능 부담을 줄였습니다.

구조적 문제

기능 나열 중심의 구성으로 방문자가 공감하기 전에 제품 설명이 먼저 등장합니다. 랜딩페이지에서 가장 중요한 "내 문제를 해결해줄 수 있나"에 대한 답 없이 스크롤이 시작되어 이탈률이 높아지는 구조였습니다.

공감 → 솔루션 → 신뢰 → 행동 순으로 섹션 흐름 재설계

반복되는 패턴

아이콘 8개 × 3섹션이 동일한 레이아웃으로 반복되어 스크롤 피로도가 높습니다. 섹션 간 배경색 변화가 없어 어디까지 읽었는지 인지하기 어려운 상태였습니다.

탭 UI + Bento Grid로 통합, 배경색 3단계 전략 적용

기술적 개선점

jQuery 의존, vh 단위 혼용, heading 계층 불일치, 접근성 속성 부재 등 코드 품질 개선이 필요한 상태였습니다.

Vanilla JS 전환, clamp() 재계산, 시맨틱 마크업 및 접근성 속성 정비
