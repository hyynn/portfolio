Duration: 2025.06 (집중작업 4일 + 리팩토링 2일)
Role: UI/UX Design, Web Publishing 100%
Tool: Figma, Whisk AI, Nanobanana, Gemini
Tech Stack: HTML, CSS, Vanilla JavaScript
Pages: Main + 3 Sub pages
Responsive: Desktop / Tablet / Mobile

Project Workflow

01

Discovery

  • 제약회사 웹사이트 레퍼런스 분석
  • 기업소개, 허가제품, ESG 중심의 페이지 구조 기획
02

Design

  • 신뢰성과 전문성을 표현하는 네이비 블루 계열 컬러 팔레트 선정
  • Whisk AI로 생성한 실험실/연구 현장 이미지 활용
  • Figma로 와이어프레임 및 UI 디자인
03

Development

  • fetch API로 header/footer를 별도 HTML 파일로 분리해 모든 페이지에서 공통 로드
  • IntersectionObserver로 스크롤 진입 시 섹션 페이드인 애니메이션 구현
  • IntersectionObserver로 카드 순차 등장(0.15초 간격) 애니메이션 구현
  • 숫자 카운터 애니메이션 구현 (0 → 100, setInterval 기반)
  • 시맨틱 table로 허가제품 정보 구조화 (colgroup, scope 적용)
  • Google Maps iframe 삽입, loading="lazy"로 초기 로딩 최적화
04

Refactoring

  • CSS 변수(:root) 도입으로 컬러 중앙 관리
  • clamp()로 반응형 폰트 사이즈 적용 (고정값 제거)
  • 브레이크포인트 4단계 반응형 레이아웃 구현 (1400/1200/900/600px)
  • 모바일 햄버거 메뉴 및 mobile-login 추가
  • section1/2 공통 선택자 통합으로 CSS 중복 제거
  • 뉴스 리스트 flex → CSS Grid 변환으로 줄바꿈 문제 해결
  • 시맨틱 마크업 개선 (section, main, article, nav, figure)
  • strong/em → h2/p 태그로 교체, div.btn → button 태그 전환
05

QA

  • 크로스 브라우징 테스트 (Chrome, Safari)
  • 반응형 검증 (데스크탑 / 태블릿 / 모바일)
06

Deploy

  • GitHub Pages 배포

Design Decisions

주요 디자인 결정과 그 이유

왜 블루 컬러인가?

#032753의 네이비 블루로 신뢰감과 권위 있는 느낌을 주고, #ACBFFF의 밝은 블루로 포인트를 주어 딱딱하지 않은 균형감을 만들었습니다. 의료/제약 분야에서 신뢰성을 전달하는 데 가장 적합한 컬러 조합입니다.

Whisk AI 이미지 활용

실제 실험실, 연구시설, 제조 현장 이미지를 Whisk AI로 생성해 활용했습니다. 제약회사의 R&D 역량과 전문성을 시각적으로 전달하기 위한 선택입니다.

Header/Footer 분리

fetch API로 header/footer를 별도 HTML 파일로 분리해 모든 페이지에서 공통으로 불러오도록 구현했습니다. 메뉴 수정 시 한 파일만 수정하면 전체 페이지에 반영됩니다.

스크롤 애니메이션

IntersectionObserver로 스크롤 진입 시점에만 애니메이션이 실행되도록 구현했습니다. 카드는 0.15초 간격으로 순차 등장하고, 섹션은 페이드인 처리해 자연스러운 흐름을 만들었습니다.

CSS 변수 & clamp() 도입

리팩토링 과정에서 :root CSS 변수로 컬러를 중앙 관리하고, 폰트 사이즈에 clamp()를 적용해 브레이크포인트별 font-size 재선언을 제거했습니다. 뷰포트에 따라 자연스럽게 스케일되어 유지보수 포인트가 줄었습니다.

Visual Section

기업 슬로건과 CTA 버튼을 배치했습니다. 배경 이미지는 Whisk AI로 생성한 연구 현장 이미지를 사용했습니다. 텍스트는 페이지 로드 시 오른쪽에서 슬라이드인되는 CSS 애니메이션을 적용했습니다.

Section 1 & 2

핵심 사업 영역(Section1)과 주요 의약품(Section2)을 카드 형태로 제시했습니다. IntersectionObserver로 스크롤 진입 시 카드가 순차적으로 등장하고, hover 시 이미지 확대 효과를 CSS transition으로 구현했습니다. 두 섹션은 동일한 구조를 공통 선택자로 통합 관리합니다.

Section 3

뉴스/공지사항을 2컬럼으로 구분하고, 전체 배경을 이미지로 처리해 섹션을 시각적으로 구분했습니다. 리스트는 CSS Grid로 날짜와 제목 열을 고정해 줄바꿈 시에도 정렬이 유지됩니다. hover 시 해당 기사가 강조되도록 CSS 효과를 적용했습니다.

Section 4

Google Maps를 iframe으로 삽입하고 loading="lazy"를 적용해 해당 섹션 진입 시점에 지도가 로드되도록 했습니다. 지도 하단 오버레이에 위치 안내 텍스트와 문의하기 버튼을 배치했으며, 버튼은 이미지 대신 CSS로 직접 구현했습니다.

Header & Footer

fetch API로 header/footer를 별도 파일로 분리해 모든 페이지에서 공통 로드되도록 구현했습니다. 현재 페이지에 맞는 GNB 메뉴가 자동으로 활성화됩니다. 모바일에서는 햄버거 메뉴와 로그인/채용/고객지원 버튼이 헤더에 상시 노출됩니다.

All Sections

Mission, Vision을 각각 풀스크린 이미지 위 텍스트 오버레이로 구성했습니다. 스크롤 시 IntersectionObserver로 해당 섹션 진입을 감지해 텍스트 박스가 페이드인되도록 구현했습니다.

Statistics Section

"100여 개국"의 글로벌 허가 실적을 숫자 카운터 애니메이션으로 표현했습니다. IntersectionObserver로 섹션 진입 시점에 setInterval 기반 카운터(0 → 100)가 실행되고, 한 번만 동작하도록 animated 클래스로 중복 실행을 방지했습니다.

Product Visual

바이오의약품, 케미컬의약품, 제품소식 카테고리를 상단 메뉴로 배치하고, 현재 활성화된 메뉴에 .on 클래스를 적용해 시각적으로 구분했습니다.

Product Table

시맨틱 table로 제품명, 이미지, 성분명, 적응증, 글로벌 허가기관 정보를 구조화했습니다. colgroup으로 열 비율을 고정하고 scope 속성으로 접근성을 개선했습니다. 모바일에서는 table-wrap에 overflow-x: auto, 테이블에 min-width를 적용해 가로 스크롤로 전체 정보를 확인할 수 있도록 했습니다. 바이오시밀러/바이오신약은 탭 버튼으로 구분했습니다.

ESG Report Cards
