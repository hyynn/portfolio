Design Decisions

주요 디자인 결정과 그 이유

왜 #A5B2A2 컬러인가?

'당신의 색감'이라는 브랜드 컨셉과 제주의 자연(바다와 숲 사이의 색)을 표현. 따뜻하면서도 차분한 느낌으로 브랜드의 감성을 담았습니다.

모바일 반응형 작업

웨딩 관련 문의는 대부분 모바일에서 이루어질 것으로 예측하고, 모바일 접근성을 고려하여 반응형 웹으로 구축했습니다.

갤러리 중심 레이아웃

스냅/영상 전문 업체의 특성상 비주얼이 가장 중요한 요소이므로, 사진이 돋보이도록 갤러리를 메인으로 배치했습니다.

클라이언트 자체 업데이트 구조

portfolio.json의 숫자만 변경하면 코드 수정 없이 사이트에 자동 반영되도록 설계해, 클라이언트가 직접 콘텐츠를 관리할 수 있도록 했습니다.

Visual

requestAnimationFrame으로 최적화된 scroll 핸들러를 사용해 hero에 클래스를 추가하고, CSS transition으로 영상이 90% 크기로 줄어들며 border-radius가 적용되도록 구현했습니다. 모바일에서는 이 효과를 비활성화해 전체 화면 몰입감을 유지했습니다.

Section1

mouseenter 이벤트로 호버한 썸네일의 이미지 src를 큰 이미지 영역에 반영하고, setInterval(3초)로 자동 롤링되도록 구현했습니다. 호버 시 인터벌을 일시 중단하고 mouseleave 시 재개하며, 전환 시 fadeOut/fadeIn 효과로 사진이 찍히는 듯한 느낌을 표현했습니다.

Section2

섹션의 offset().top을 변수에 캐싱하고 scroll 위치와 비교해 뷰포트 진입 시점에 클래스를 토글하는 방식으로 구현했습니다. 트리거 기준점은 모바일(winH * 0.7)과 데스크탑(winH / 2)으로 분기 처리했습니다.

Section3

Swiper의 loop: true, autoplay 옵션으로 무한 자동재생을 구현하고, pagination과 navigation 버튼을 함께 적용했습니다.

Header & Footer

인스타그램 채널과 카카오 오픈채팅 바로가기는 헤더의 GNB에 포함해 모든 페이지에서 접근할 수 있도록 구성했습니다. 푸터에는 연락처와 예약문의 정보를 배치했습니다.

Header

포트폴리오 섹션은 업체의 특성에 맞춰 갤러리를 중심으로 구성했습니다. 메인과 동일한 브랜딩을 유지하면서도 갤러리에 최적화된 레이아웃을 적용했습니다.

Gallery Section

portfolio.json에서 파일 개수를 읽어 이미지를 로드하고, 새로고침 시 비율(세로/가로)을 랜덤 배치해 매번 다른 레이아웃이 연출되도록 구현했습니다. 라이트박스 기능으로 클릭 시 원본 이미지를 큰 화면으로 감상할 수 있습니다.

Navigation

야외스냅, 야외필름, 본식스냅, 본식필름 4개 카테고리로 분류했습니다. 스냅(이미지)과 필름(영상)이 혼재하는 구조라 이미지는 <img>, 영상은 <video>로 각각 별도 함수로 생성하고, 영상에는 데스크탑 hover 시 미리보기 재생(터치 기기에서는 비활성화)과 라이트박스 연동을 추가했습니다.

Introduction Section

Brand Story는 업체 소개와 추구하는 방향성에 관한 정보를 담고 있습니다. About/Vision으로 섹션을 나누고, 각 페이지에서는 스토리텔링을 통해 브랜드의 가치와 철학을 담을 수 있도록 구성했습니다.

User Experience

소개 페이지 내 메뉴 클릭 시 페이지 최상단으로 이동하지 않고, 현재 스크롤 위치를 유지한 채 콘텐츠만 교체되도록 구현했습니다.

URL Parameter

GNB의 Vision 링크에 ?tab=vision 파라미터를 부여해 외부에서 직접 특정 탭으로 진입할 수 있도록 구현했습니다. URLSearchParams로 파라미터를 파싱해 해당 탭을 활성화합니다.

Main

모바일에서는 hero 영상 축소/라운드 효과를 비활성화하고, Section1의 썸네일 hover 롤링도 비활성화했습니다. 대신 자동 롤링만 유지해 터치 환경에서 자연스럽게 작동하도록 처리했습니다.

Sub Page 1

GNB 하위 메뉴(lnb)를 클릭 시 slideToggle로 펼치고, 다른 메뉴 클릭 시 나머지는 slideUp으로 닫히도록 구현했습니다. 메뉴가 열릴 때 헤더에 menu-open 클래스를 추가해 position: fixed로 전환하고, 화면 밖 클릭 시 메뉴가 닫히도록 document click 이벤트를 추가했습니다.

Sub Page 2

갤러리 이미지 비율을 모바일 기준(세로형 위주)으로 별도 분기 처리하고, IntersectionObserver로 뷰포트 진입 시점에만 이미지를 로드하는 lazy loading을 적용했습니다.
